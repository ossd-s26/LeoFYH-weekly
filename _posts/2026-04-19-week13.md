**Week13 Reflection**
This week, I explored several open-source projects beyond code-focused tools. One project I found interesting is Open Library by the Internet Archive. It aims to create a web page for every book ever published and provides free access to millions of books. I think projects like this are very meaningful because they make knowledge more accessible to everyone, especially people who cannot afford books or do not have access to libraries.

At the same time, I continued contributing to the Nanobot project. This week, we started working on adding new features, which helped me better understand how real-world open-source projects evolve over time. Feature development requires not only coding skills but also communication and alignment with the existing system.

In addition, while working with the codebase, I identified a small issue and submitted a pull request to fix it: https://github.com/HKUDS/nanobot/pull/3178
. This experience made me realize that contributing to open source does not always mean building big features. Even small fixes can improve the project and help other users.

Overall, this week helped me see open source from both a social impact perspective and a practical development perspective.
